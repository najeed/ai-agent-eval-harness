# STORY: The Concurrent Blitz

The clock is ticking toward midnight. By dawn, three different agent fleets—one in the cloud (`http`), one on a local secure server (`local`), and one in a legacy data-center (`socket`)—must be cleared for production.

"We can't run them one-by-one," the Operations Director says, pacing the room. "If we do, we won't finish until next Tuesday. We need a Blitz."

You look at the dashboard. "A Blitz means concurrency. It means running fifty trials at once, spanning all three protocols. If an agent hangs, we retry. If a connection drops, we fail-over."

"Can the harness handle that much pressure?"

"It was built for this," you reply. "We don't just 'evaluate'—we orchestrate. Today, you'll manage the 'Mixed-Fleet' and prove that we can benchmark at scale without breaking a single pipe."
