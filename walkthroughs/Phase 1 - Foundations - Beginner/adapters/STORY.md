# STORY: The Multilingual Agent

"We hear they have an agent that speaks only binary," the Chief Communications Officer says, looking at a secure dossier. "But our evaluations expect a structured JSON response over HTTP. We're in a bit of a bind."

"That's because a harness is like a translator," you say, taking the first step into the secure lab. "To evaluate these different engines, we have to build the **Bridge** between them."

You point to a schematic labeled **'Adapter Framework'**. "One agent might live on a remote cloud server (`http`), another might be a local script residing in this repo (`local`), or even a persistent stream (`socket`). It doesn't matter. To us, they all speak evaluation."

Your task today is to build the "Bridge." You'll configure the harness to talk to a local Python script instead of a remote server and prove that you can evaluate any agent, anywhere, regardless of its 'native tongue.'
