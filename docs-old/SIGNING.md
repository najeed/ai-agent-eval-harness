# Guide: Trust Protocol & Cryptographic Signing (ED25519)

The **Trust Protocol** establishes a non-repudiable audit trail for AI agent evaluations. By using asymmetric cryptography (ED25519), traces generated by the harness can be "sealed" and later verified to ensure they haven't been tampered with and that they genuinely originate from a trusted environment.

---

## 1. Key Concepts

- **Private Key**: Used by the Evaluation Runner (CI/CD) to sign the final `run.jsonl` trace.
- **Public Key**: Shared with stakeholders to verify the signature.
- **Behavioral Fingerprint (VC)**: A JSON certificate (`fingerprint.json`) that snapshots the scenario's topology and tool definitions. This ensures the agent wasn't tested against a "weakened" version of the scenario.
- **Hard Gate**: A CI/CD step that fails if the trace's signature is invalid or if the agent failed the evaluation.

---

## 2. Key Management

The `TraceVerifier` manages keys in the `~/.aes/keys/` directory by default.

### Generating Keys
You can generate a new ED25519 key pair using the internal utility or the CLI:

```python
from eval_runner.verifier import TraceVerifier
verifier = TraceVerifier()
private_key, public_key = verifier.generate_key_pair()
# Keys are saved to ~/.aes/keys/private_key.pem and public_key.pem
```

> [!WARNING]
> Keep your **Private Key** secure. If it is compromised, unverified traces can be signed as "trusted."

---

## 3. Signing a Trace

Once an evaluation completes, the harness can sign the `run.jsonl` file using your private key. This appends a cryptographic signature to the end of the trace metadata.

```bash
# Internal signature generation (managed by the engine during run)
multiagent-eval evaluate --path scenarios/finance --sign
```

---

## 4. CI/CD Gatekeeping (`gate`)

The `gate` command is designed for use in GitHub Actions, GitLab CI, or Jenkins. It returns a non-zero exit code if verification fails.

### Basic Integrity Gate (SHA-256)
Verifies that the trace ended in `SUCCESS` and its internal hashes match:
```bash
multiagent-eval gate --path reports/run.jsonl
```

### Full Cryptographic Gate (ED25519)
Verifies the signature against a public key and matches it to a specific scenario fingerprint:
```bash
multiagent-eval gate \
  --path reports/run.jsonl \
  --vc scenarios/finance/fingerprint.json \
  --public-key keys/prod_public.pem \
  --hash ${{ github.sha }}
```

---

## 5. Integration Example: GitHub Actions

```yaml
jobs:
  agent-eval:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Run Evaluation
        run: multiagent-eval evaluate --path scenarios/production/
      
      - name: Trust Protocol Gate
        run: |
          multiagent-eval gate \
            --path runs/run.jsonl \
            --hash ${{ github.sha }}
```

---

## 6. Verification Failures

If `gate` fails, the engine will output detailed failure reasons:
- **`SIGNATURE_MISMATCH`**: The trace was modified or signed with a different key.
- **`FINGERPRINT_DIVERGENCE`**: The scenario topology at test-time doesn't match the sanctioned fingerprint.
- **`TRACE_FAILURE`**: The agent failed one or more critical tasks.
- **`HASH_MISMATCH`**: The trace metadata does not match the provided Git commit hash.
